• ¿Qué tipo de error crees que es más común en programas reales?

El error más común que yo creo en programas reales es el error de entrada de datos del usuario (ValueError), ya que los usuarios pueden ingresar información incorrecta como letras en campos numéricos, caracteres especiales,valores vacíos o formatos inválidos.

• ¿Por qué es importante manejar excepciones?

Manejar excepciones es importante porque permite que el programa evite cierres inesperados, entregue mensajes de error claros y comprensibles al usuario. Además, mejora la estabilidad y confiabilidad del sistema, ya que permite controlar situaciones inesperadas sin afectar el funcionamiento general de la aplicación. El uso de excepciones también ayuda a separar la lógica principal del manejo de errores, facilitando el mantenimiento del código y aumentando su calidad profesional.
