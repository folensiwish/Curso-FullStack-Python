1. El rol de una base de datos

Una base de datos relacional cumple el rol de almacenar, organizar y proteger la información de una empresa u organización. Permite que los datos estén disponibles de forma rápida, segura y ordenada, facilitando el trabajo de los sistemas y aplicaciones que dependen de esa información.
    Ejemplos: 
        -Sistema de recursos humanos: almacenar datos de empleados, cargos, sueldos y asistencia.
        -Sistema académico: registrar estudiantes, cursos, notas y matrículas.
        -Sistema de reservas: gestionar reservas de hoteles, vuelos, citas médicas o arriendos.

2. Características de un RDBMS

Un RDBMS es un sistema que permite crear, administrar y consultar bases de datos relacionales utilizando tablas relacionadas entre sí.
    Características principales:
        -Uso de tablas relacionadas: organiza la información en filas y columnas.
        -Soporte para SQL: permite consultar y modificar datos mediante lenguaje SQL.
        -Integridad de datos: usa llaves primarias y foráneas para mantener relaciones correctas.
        
Base de dato relacionales más utilizados en la industria:
    -PostgreSQL: usado en aplicaciones web, sistemas empresariales y proyectos backend por su estabilidad y funciones avanzadas.    
    -MySQL: muy usado en sitios web y sistemas pequeños o medianos. 
    -Oracle Database: utilizado en empresas grandes y sistemas corporativos de alto volumen de datos.

3. Herramientas y objetos
Herramientas para consultar bases de datos. Algunas herramientas que se pueden utilizar son:
    -DBeaver: herramienta gráfica que permite conectarse, consultar datos y administrar bases de datos de forma visual.
    -psql: herramienta de línea de comandos para PostgreSQL que permite ejecutar consultas directamente desde la terminal.

3.1. Objetos de una base de datos:
    -Tabla : Es el objeto principal donde se almacenan los datos. Está formada por filas (registros) y columnas (campos).
    -Vista: Es una consulta guardada que muestra datos de una o varias tablas. No almacena datos físicos, solo presenta información de forma organizada.
    -Índice: Permite acelerar las búsquedas dentro de una tabla. Funciona de forma similar al índice de un libro, facilitando encontrar información rápidamente.
    -Primary key: Es un campo que identifica de forma única cada registro de una tabla. No puede repetirse ni ser nulo.
    -Foreign Key: Es un campo que crea una relación entre dos tablas. Permite vincular registros y mantener la integridad referencial.
